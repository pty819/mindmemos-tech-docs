1. 项目概述

1.1. 一句话定位

MindMemOS 是一个给 AI Agent 用的长期记忆系统，核心能力是在对话/交互过程中持续写入记忆、检索记忆、离线巩固记忆、以及从反馈中学习。

1.3. 文档定位

本文档不是 API 参考或函数文档。它从模块设计角度回答三个问题：

  1. 模块之间怎么穿起来的 — 各 package / subpackage 的依赖关系和调用链

  2. 关键功能的时序视图 — 一个请求从收到到写完，经过了哪些模块、谁先谁后

  3. 核心设计决策 — 为什么某些模块是独立的、为什么选这种互联方式

全文档统一使用 Mermaid 图来描述模块互联和时序，代码中只引用模块/类/接口级别的符号，不深入到具体函数的参数和返回值。

1.4. 五层架构速览

从代码目录结构来看，系统自然地分为五层：

层

目录

职责

API 层

api/

HTTP 入口、认证、序列化、路由

Pipeline 层

pipelines/

业务流程编排，通过 Registry 插拔

Component 层

components/

无状态算法组件，被 Pipeline 调用

Infra 层

infra/

数据库/Kafka 驱动，业务无关

Foundation 层

typing/, config/, llm/, mappers/

全系统共享的 DTO、配置、LLM 客户端

这五层不是严格分层架构（上层不依赖下层），而是一个依赖倒置的模块级别体系——typing/ 被所有模块导入，infra/ 只被 pipelines/memory_db/ 调用。具体互联方式见下一章。
